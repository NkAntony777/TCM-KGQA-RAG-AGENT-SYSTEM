# 问答路由与评测基线更新

更新时间：2026-04-06

## 本轮目标

- 将多数实际问答默认路由调整为 `hybrid`
- 修复高难学术问题被医疗边界误拒的问题
- 避免检索模式在复杂问题上乱召回无关实体
- 为后续 `quick / deep / agent` 继续演进打稳评测基线

## 已完成变更

### 1. 默认路由策略调整为 `hybrid`

- 当前策略不再把“带实体的出处 / 原文 / 定义 / 文献记载”默认判为 `retrieval`
- 含明确图谱实体锚点的问题，默认优先进入 `hybrid`
- 仅以下问题仍优先保持 `graph`
  - 方剂组成
  - 药材归经性味
  - 症状到证候
  - 证候到方剂
  - 明确路径查询

### 2. 查询理解增强

- `TCMIntentClassifier` 已新增运行时图谱实体词典加载
- 已加入方剂后缀正则识别，如 `丸 / 散 / 汤 / 饮`
- 已加入可选 `jieba` 分词增强
- 对长问题和跨学科问题，能更稳定地抽出核心实体锚点

### 3. 检索乱召回抑制

- `retrieval_service` 已新增 lexical sanity gate
- 当召回文档不包含查询中的核心实体锚点时，会被过滤
- 现场验证中，五苓散高难题已不再乱召回柴胡、逍遥散等无关证据

### 4. 医疗边界误拒修复

- 原先“剂量 / 用量”相关字样会误伤学术讨论类问题
- 当前已区分：
  - 可执行的个人服药 / 开方 / 频次建议：继续拒答
  - 学术机制、阈值效应、煎煮法、AQP 等研究型问题：允许进入问答链路

### 5. 启动脚本清理旧进程

- 启动脚本已统一复用 `scripts/start_uvicorn_service.ps1`
- 启动前会只清理当前项目对应端口和模块的旧服务残留
- 避免重复监听、假死和端口占用问题

## 评测基线同步

已同步修改以下评测集，使其符合新的产品语义：

- `backend/eval/datasets/router_smoke_20.json`
- `backend/eval/datasets/graph_qa_20.json`

调整原则：

- 绝大多数带实体的出处 / 原文 / 定义 / 文献类问题，`expected_route` 从 `retrieval` 改为 `hybrid`
- 评测重点从“是否走纯 retrieval”转为“是否保持图谱锚点、是否能正确补文献证据”

## 当前结论

- `quick` 模式的默认入口已经更适合走 `hybrid`
- 纯 `retrieval` 不再适合作为大多数实体问答的默认路由
- 后续 `deep` / `agent` 的重点应放在：
  - 更智能的 planner
  - 多轮补证据
  - 更强的出处抽取
  - 更好的可视化 trace
