# 优化成功性消融实验设计

更新时间：2026-04-14  
适用子项目：`langchain-miniopenclaw/backend`

## 1. 文档目的

本文档用于为本项目当前已经完成的核心优化设计一组**可复现实验、可写入论文、可用于答辩或中期/终期汇报**的消融实验方案。

设计目标有三点：

1. 证明当前优化不是“主观感觉变好”，而是在**准确性、稳定性、时延、可解释性**上有可测收益。
2. 让实验能够直接对应当前已实现的代码开关、评测脚本与运行模式。
3. 让后续论文写作时，可以将实验结果自然组织为“方法有效性验证”与“工程优化收益验证”。

## 2. 总体原则

所有消融实验尽量遵循以下原则：

### 2.1 单变量控制

每组实验只改变一个关键因素，其他条件保持一致，包括：

- 相同的数据集
- 相同的 `top_k`
- 相同的后端部署方式
- 相同的 LLM provider
- 相同的评测时间窗

### 2.2 至少记录四类指标

每组实验至少记录：

1. **质量指标**
   - `answer_hit`
   - `book/source hit`
   - `predicate hit`
   - `专家评分` 或 `代理评分`
2. **性能指标**
   - `avg_latency_ms`
   - `p95_latency_ms`
3. **稳定性指标**
   - `request_error_rate`
   - `fallback_rate`
   - `degraded_rate`
4. **解释性/结构指标**
   - `route distribution`
   - `graph-only / hybrid / retrieval-only 占比`
   - `tool_count`
   - `citation completeness`

### 2.3 结果要分成两层

建议所有实验结果分成两层：

- **Retrieval 层结果**
- **End-to-End QA 层结果**

因为有些优化主要提升检索，而不一定直接提升答案长度；反之也一样。

## 3. 建议重点消融实验矩阵

建议重点做 8 组实验，其中前 5 组最关键。

## 实验 A：向量化链路 vs 非向量化主链

### A1. 实验问题

当前项目最大的架构变化之一，是从 `vector-first` 退居为兼容层，转向：

`graph + files-first + structured index`

因此最核心的验证问题是：

> 非向量化主链是否在中医知识问答中，比向量化主链更准确、更稳定、且不显著恶化时延？

### A2. 对照组

1. **Baseline: 向量化兼容链路优先**
   - 开启：
     - `RETRIEVAL_VECTOR_COMPATIBILITY_ENABLED=true`
     - `FILES_FIRST_DENSE_FALLBACK_ENABLED=true`
     - `CASE_QA_VECTOR_FALLBACK_ENABLED=true`
   - 并尽量让 retrieval 更容易走 dense / hybrid fallback

2. **Experimental: 非向量化主链**
   - 关闭：
     - `RETRIEVAL_VECTOR_COMPATIBILITY_ENABLED=false`
     - `FILES_FIRST_DENSE_FALLBACK_ENABLED=false`
     - `CASE_QA_VECTOR_FALLBACK_ENABLED=false`

### A3. 数据集

建议使用三类数据集：

1. [qa_origin_source_probe_4.json](D:/毕业设计数据处理/langchain-miniopenclaw/backend/eval/datasets/qa_origin_source_probe_4.json)
2. [qa_agent_hard_probe_10.json](D:/毕业设计数据处理/langchain-miniopenclaw/backend/eval/datasets/qa_agent_hard_probe_10.json)
3. 博士级 10 题 live probe

### A4. 指标

建议记录：

- `docqa failed count`
- `complexqa failed count`
- `book_hit_rate`
- `citation completeness`
- `request_error_rate`
- `avg_latency_ms`
- `fallback_rate`

### A5. 预期结论

预期应证明：

- 非向量化主链在出处题、条文题、比较题上显著更稳
- 向量链路在这些题目上召回不够可控
- 非向量化主链虽然不一定每题都更快，但整体更可靠、更可解释

## 实验 B：图谱优先 vs 文本优先

### B1. 实验问题

当前项目已将图谱从“约束器”提升为“显式证据源”，但仍需证明：

> 在关系型问题上，graph-first 是否比直接 files-first / retrieval-first 更优？

### B2. 对照组

1. **Baseline: 关闭 graph-first 优先**
   - 强制关系题走 `hybrid/retrieval`

2. **Experimental: graph-first**
   - 采用当前策略层：
     - 组成题 → graph
     - 功效题 → graph
     - 主治题 → graph
     - 比较题在必要时再 hybrid

### B3. 数据集

建议构建一个小型关系题集合：

- 方剂组成
- 功效
- 治法
- 归经
- 主治
- 推荐方剂

可以从现有博士题中抽取：

- q01
- q02
- q03
- q09
- q10

### B4. 指标

- `predicate_hit_rate`
- `structured_evidence_presence`
- `final_route=graph 占比`
- `avg_latency_ms`
- `answer completeness`

### B5. 预期结论

预期应证明：

- graph-first 更适合关系型中医问题
- 可以减少不必要全文检索
- 可提高结构证据覆盖率

## 实验 E：Deep hardening 前后对比

### E1. 实验问题

Deep 模式曾出现：

- `planner_deterministic_fallback`
- 极短答案
- 瞬时 provider 异常导致完整分析失败

因此需要证明：

> Deep hardening（重试、超时放宽、deep->quick grounded fallback）是否显著提升了 Deep 模式的稳定性与质量下限？

### E2. 对照组

1. **旧版**
   - 无 retry
   - 直接 deterministic fallback

2. **新版**
   - retry
   - extended timeout
   - deep -> quick grounded fallback
   - generation diagnostics

### E3. 数据集

建议直接用博士级 10 题。

### E4. 指标

- `deep_ok`
- `deep_fallback_count`
- `deep_short_answer_count`
- `planner_deterministic_fallback_count`
- `avg_latency_ms`
- 专家评分

### E5. 预期结论

预期应证明：

- 新版显著减少 Deep 崩塌式短答
- Deep 质量下限提高
- 时延可能略有波动，但稳定性收益显著

## 4. 推荐论文主实验组合

如果论文篇幅有限，建议只突出以下 5 组：

1. **向量化链路 vs 非向量化主链**
2. **图谱优先 vs 文本优先**
3. **SQLite path vs Nebula path**

## 6. 实验写作建议

写论文时建议将消融结论表述为：

### 6.1 架构层结论

非向量化主链在中医知识问答中显著提升了出处定位、章节约束和关系型问题的检索稳定性。

### 6.2 图谱层结论

图谱优先策略对于关系型问题具有更高的结构证据覆盖率，并减少了不必要的全文检索。

### 6.3 检索增强层结论

LLM section summary 可以提升 section 级命中与 QA 端到端命中质量，说明结构化摘要缓存具有真实收益。

### 6.4 工程层结论

Deep hardening、schema migration 与 strict release gate 降低了生产链路中的失败率和误放行风险。

## 8. 总结

如果目标是“突出我们优化的成功性”，最关键的不是堆很多实验，而是让实验能清楚回答这三个问题：

1. 当前架构是否真的比旧架构更好？
2. 这些改动是否真的带来了准确性/稳定性/时延收益？
3. 收益是否能直接归因到具体优化，而不是随机波动？

本文档给出的消融实验设计，就是围绕这三个问题展开的。
