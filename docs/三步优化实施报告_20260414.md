# 三步优化实施报告

更新时间：2026-04-14  
适用子项目：`langchain-miniopenclaw/backend`

## 1. 本轮目标

基于当前主链已经完成的去向量化收口，本轮继续推进三项高价值优化：

1. 图谱证据升级
2. 结构感知 files-first
3. 受控单轮 query refinement

目标不是引入新型 agent 花样，而是在不破坏现有 `graph + files-first + structured index` 主链的前提下，进一步提升：

- 关系型问题的直答效率
- 章节级命中能力
- 低命中查询的恢复能力

## 2. 优化一：图谱证据升级

### 2.1 目标

让图谱关系不再只充当“去哪里读原文”的导航器，而是升级为答案中的一等证据源。

具体目标：

- 关系型问题优先依据 graph relation 组织结论
- 出处型问题仍优先补原文/章节证据
- 答案中显式区分“结构化图谱证据”和“文献原文证据”

### 2.2 实施内容

本轮实现了以下改动：

- 在 `retrieval_strategy` 中新增 `answer_policy`
  - `graph_relation_first`
  - `graph_relation_with_origin`
- 对以下意图默认启用图谱优先回答策略：
  - `formula_composition`
  - `formula_efficacy`
  - `formula_indication`
  - `syndrome_to_formula`
- 对以下意图启用“图谱 + 出处补证”策略：
  - `formula_origin`
  - `compare_entities`

同时，在答案构造层新增事实证据分组：

- `structured`
- `documentary`
- `other`

并将其写入：

- 最终响应
- `evidence_bundle`
- grounded prompt

### 2.3 结果

当前回答层已能够：

- 先使用图谱关系组织关系型答案
- 再补充对应的原文/出处证据
- 避免把所有证据混成单一列表塞给 LLM

这使“结构 + 出处”的双证据表达成为默认能力。

## 3. 优化二：结构感知 files-first

### 3.1 目标

在现有 leaf chunk + parent 回读的基础上，进一步减少“只能先命中 chunk，再被动回整章”的依赖，增强 section 级的直接命中能力。

具体目标：

- 为 section 预生成摘要与标签
- 新增 section 级 FTS
- 让搜索结果可以直接返回章节级命中，而不总是先返回 leaf

### 3.2 实施内容

在 `files_first_support.py` 中新增：

- `section_summary`
- `topic_tags`
- `entity_tags`
- `representative_passages`

并扩展了本地索引结构：

- `docs` 增加结构字段
- `docs_fts` 增加结构字段
- 新增 `sections`
- 新增 `sections_fts`

本轮采用的是**启发式结构感知**，而不是额外引入摘要模型：

- `section_summary` 由章节正文前部压缩得到
- `topic_tags` 由领域关键词与章节标题抽取
- `entity_tags` 由方剂实体模式抽取
- `representative_passages` 由章节中前若干高信息句抽取

另外做了两项关键收口：

- `chapter://` 读取时直接返回 section 摘要、标签和代表性段落
- 如果 `sections_fts` 未单独命中，但 leaf 明显落在同一节，搜索层会自动合成 section 结果，避免表现退回纯 leaf 视角

### 3.3 结果

当前 files-first 已从“leaf 命中 + 被动整章回读”升级为：

- 可显式索引 section
- 可直接返回章节级候选
- 可携带摘要/标签/代表性片段

这为后续的章节摘要检索、结构感知重排、论文方法展示提供了更扎实的基础。

## 4. 优化三：受控单轮 query refinement

### 4.1 目标

在 files-first 低命中时，增加一次轻量级的查询改写机会，但不能把主链改造成开放式的 agentic iterative retrieval。

具体原则：

- 只触发一次
- 只在低命中时触发
- 仍回到 graph/files-first 主链
- 不默认转入 dense 主路径

### 4.2 实施内容

在 `RetrievalEngine.search_hybrid(...)` 中新增：

- `_maybe_refine_files_first_query(...)`
- `_refine_files_first_query(...)`
- `_prefer_refined_result(...)`

触发条件为：

- `search_mode == files_first`
- 无结果或仅极少结果
- 或首条分数过低
- 或出现 `files_first_sparse_query_empty`

改写优先级：

1. 若 LLM 改写可用，则生成一条更适合古籍/图谱检索的短查询
2. 若 LLM 不可用，则退回 alias 扩展型启发式改写

改写后只重跑一次 `run_hybrid_search(...)`，并通过结果对比决定是否采纳 refined result。

### 4.3 结果

当前系统已经具备：

- files-first 低命中自动自救
- 单轮闭环，不会无限迭代
- 改写成功时保留 `refined_query`
- 改写无收益时保留原结果并标注 `single_query_refinement_no_gain`

这使查询恢复能力增强，但没有破坏主链的稳定性和可解释性。

## 7. 当前收益

本轮完成后，系统相较于上一状态的直接收益主要有：

1. 关系型问题的答案表达更合理
   - 图谱关系已成为显式证据层
   - 不再把 graph 仅当作原文导航器

2. files-first 的章节级命中能力增强
   - 支持 section 级摘要/标签
   - 支持 section 级直接结果
   - 减少整章回读的粗粒度开销

3. 低命中查询具备一次性恢复能力
   - 避免 files-first 一次没命中就结束
   - 又不会回到默认 dense-first 逻辑

## 8. 剩余可继续深化的方向

虽然本轮三项优化已完成最小可用版本，但仍有进一步深化空间：

### 8.1 图谱证据升级后续

- 对 graph relation 增加更细的答案模板
- 按谓词族进一步优化图谱证据排序
- 对“图谱结论 + 原文佐证”的配比做意图化控制

### 8.2 结构感知 files-first 后续

- 将 `section_summary / topic_tags / entity_tags` 从启发式升级为离线 LLM 生成
- 增加 section 级 rerank
- 对 `chapter://` 增加代表性条文优先读回

### 8.3 query refinement 后续

- 引入更稳定的低命中阈值判定
- 区分“出处题改写”和“比较题改写”
- 为 refinement 增加缓存，减少重复改写成本

## 9. 总结

本轮三步优化没有改变项目的总体架构方向，而是在既有主链上做了进一步收口：

- 图谱从“约束器”升级为“显式证据源”
- files-first 从“chunk 命中 + 回读”升级为“结构感知检索”
- 查询恢复从“无结果即失败”升级为“受控单轮自救”

因此，本轮工作的价值不在于增加了多少新模块，而在于：

> 让现有 `graph + files-first + structured index` 主链更加像一条成熟、闭环、可解释的产品级问答链路。
