# Predicate Normalization Rules v3

> 用途：本文件是当前中医知识图谱谓词归一化的第三版审核规则。  
> 原则：本文件只定义“允许做什么”和“绝对不能做什么”，不直接修改图数据。  
> 执行要求：任何物理改写前，都必须先通过值域检查、类型分布检查和样例抽检。  
> 状态：`reviewed-but-not-applied`

## 1. 总体原则

1. 只有在**语义等价**、**subject/object 类型分布一致**、**下游用途一致**时，才允许做物理改写。
2. 语义相关但粒度不同的谓词，只允许归入同一“关系族”，**不允许直接物理改写原始谓词**。
3. 任何涉及：
   - 现代研究图谱
   - 书目/出处元数据
   - 特殊古籍语义
   - 治法短语直接提升为 predicate
   的关系，默认进入人工审核，不做自动归一。
4. 本规则优先保障图谱语义正确性，其优先级高于“减少谓词数量”。

---

## 2. 建议直接归一化（允许物理改写）

这类关系语义高度一致，仅是表述差异，可在通过值域校验后统一到目标谓词。

| 原谓词 | 目标谓词 | 风险 | 执行条件 |
| --- | --- | --- | --- |
| `表现症状` | `常见症状` | 低 | subject/object 类型分布与 `常见症状` 保持一致 |
| `相关症状` | `常见症状` | 低 | subject/object 类型分布与 `常见症状` 保持一致 |
| `药味` | `五味` | 低 | object 值域需基本稳定在酸、苦、甘、辛、咸或其标准英文五味 |

### 2.1 `药味 -> 五味` 的附加限制

仅当以下条件同时满足时才能执行：

1. subject_type 基本稳定为 `herb`
2. object_type 基本稳定为 `property`
3. object 文本不出现长句、复合描述或自由文本段落
4. 抽样样例中不混入“气味综合描述”或“现代英文说明句”

若不满足，降级为 `family_only`，不做物理改写。

---

## 3. 仅建议归入关系族（不允许物理改写）

这类关系语义有关联，但存在粒度差异、来源差异或对象空间差异。只允许在查询层、统计层、展示层挂到同一父类，不允许批量改写原始谓词。

| 原谓词 | 建议归入族 | 风险 | 说明 |
| --- | --- | --- | --- |
| `治疗证候` | `主治族` | 中 | 不能与 `治疗疾病`、`治疗症状` 物理合并 |
| `治疗疾病` | `主治族` | 中 | 同上 |
| `治疗症状` | `主治族` | 中 | 同上 |
| `常见症状` | `临床表现族` | 低 | 与 `表现症状`、`相关症状` 一起归族 |
| `表现症状` | `临床表现族` | 低 | 若未物理改写前可先归族 |
| `相关症状` | `临床表现族` | 低 | 若未物理改写前可先归族 |
| `药性特征` | `药性理论族` | 高 | 目前不能定为 `归经`，必须谨慎 |
| `药性` | `药性理论族` | 低 | 保留独立边，但可归族 |
| `归经` | `药性理论族` | 低 | 保留独立边，但可归族 |
| `五味` | `药性理论族` | 低 | 保留独立边，但可归族 |
| `升降浮沉` | `药性理论族` | 高 | 仅归族，不改写 |
| `适应证` | `适用范围族` | 高 | 不得直接归一到 `现代适应证` |
| `现代适应证` | `适用范围族` | 中 | 只做族级归并，不改原边 |

---

## 4. 必须保持独立（严禁自动归一）

这类关系涉及现代研究精细语义、文献学元数据或特殊经典语义，必须保持独立，不参与自动归一。

| 原谓词 | 操作 | 风险 | 原因 |
| --- | --- | --- | --- |
| `作用靶点` | 保持独立 | 极高 | 现代药理学核心边，`ingredient -> gene` 语义精确，不能并掉 |
| `关联靶点` | 保持独立 | 高 | 与 `作用靶点` 有关，但不等价 |
| `含有成分` | 保持独立 | 极高 | 中药材/方剂与成分的核心边，不能归并 |
| `用法` | 保持独立 | 高 | 不等于 `炮制方法`，也不应自动并入其他关系 |
| `药材基源` | 保持独立 | 高 | 语义比“拉丁学名”更宽，不能直接改写 |
| `出处` | 保持独立 | 高 | 属于文献学元数据，object 类型混杂 |
| `欲解时` | 保持独立 | 高 | 经典条文特有语义，样本少但语义独特 |
| `功效` | 保持独立 | 高 | 宾语空间混有短标签与长文本，不能直接并入别的关系 |
| `关联术语` | 保持独立 | 高 | 元数据弱关联，不应视为治疗/主治关系 |

## 5. 必须人工审核（不能自动决定）

这些谓词当前证据不足，不能直接判定归一方案。

| 原谓词 | 当前建议 | 风险 | 原因 |
| --- | --- | --- | --- |
| `药性特征` | 人工审核 | 高 | 样例显示可能混有 `channel/property` 等不同语义 |
| `利水化饮` | 人工审核 | 高 | 很可能是治法词被错误提升为 predicate |
| `理气活血` | 人工审核 | 高 | 同上 |
| `适应证` | 人工审核 | 高 | 与 `现代适应证` 仅可归族，不能贸然改写 |

---

## 6. 数据清洗专项（不属于谓词归一化本体）

这些问题应通过数据清洗或抽取规则修复，而不是靠谓词归一化解决。

| 发现项 | 问题 | 建议 |
| --- | --- | --- |
| `升降浮沉` | 存在 `property -> property` 自反或混乱数据 | 修复导入/抽取规则，保留原谓词 |
| `使用药材` | 出现大量 `formula -> herb` 之外的主体类型 | 进入 ontology 边界清洗，不做谓词归一 |
| `推荐方剂` | 存在 `disease -> formula` 等边界混杂 | 进入 ontology 边界清洗，不做谓词归一 |
| `治疗证候` | 存在 `disease -> syndrome`、`syndrome -> syndrome` 混杂 | 进入 ontology 边界清洗，不做谓词归一 |
| `source_chapter` | 大量 `book_prefixed_chapter_rows` | 做 chapter 规范化，不靠谓词层处理 |

---

## 7. 执行优先级

### 第一批：可低风险推进

1. `表现症状 -> 常见症状`
2. `相关症状 -> 常见症状`
3. `药味 -> 五味`
   前提：先做值域检查

### 第二批：只做关系族，不做数据写回

1. `主治族`
2. `临床表现族`
3. `药性理论族`
4. `适用范围族`

### 第三批：人工审核后再决定

1. `药性特征`
2. `适应证`
3. `利水化饮`
4. `理气活血`
5. `药材基源`
6. `用法`

---

## 8. 禁止事项

1. 禁止把 `作用靶点`、`含有成分`、`出处`、`欲解时` 纳入自动归一脚本。
2. 禁止把 `用法` 自动改为 `炮制方法`。
3. 禁止把 `药材基源` 自动改为 `拉丁学名`。
4. 禁止把 `适应证` 自动改为 `现代适应证`。
5. 禁止在未审核完类型分布与样例前，对高风险 predicate 做批量写库。
